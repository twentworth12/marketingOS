## Playbook 2: Fix Underperforming Campaign

**Time: 30 seconds**

```bash

./linkedin analyze CAMPAIGN_ID --recommend

# - Titles AND Skills in separate groups (too restrictive)

```

**Real example (PayPal):**
- Before: $156 spent, 9 clicks, 3 conversions ($17.37 CPC)
- Problem: Titles AND Skills (restrictive)
- Fix: Copied Chewy's skills-only approach
- Expected: 2-3x more clicks

## Troubleshooting Common Issues

### Issue: "Campaign has zero clicks"

**Diagnosis:**
```bash
./linkedin analyze CAMPAIGN_ID --recommend
```

**Common causes:**
1. Restrictive AND logic (titles AND skills in separate groups)
2. Missing critical IC roles (DevOps Engineer, Platform Engineer)
3. No creatives serving

**Fix:**
```bash
./linkedin copy-targeting --source 447062324 --target CAMPAIGN_ID
```

---

### Issue: "High CPC (>$30)"

**Diagnosis:** Targeting too broad or wrong audience

**Fix:**
1. Check seniority levels (exclude too-junior)
2. Add job functions (Engineering + IT only)
3. Use skills not just titles

---

### Issue: "Can't find campaign"

**Cause:** API returns max 100 results, campaign is beyond first page

**Fix:** Tool automatically paginates. Use search:
```bash
./linkedin list --name "PartialName"
```

---

### Issue: "Targeting AND vs OR confusion"

**Diagnosis:**
```bash
./linkedin analyze CAMPAIGN_ID
```

**Will show:**
```
⚠️ Titles and Skills are in DIFFERENT facet groups
Logic: (Title1 OR Title2) AND (Skill1 OR Skill2)
Effect: RESTRICTIVE - person needs BOTH
```

**Fix:** Copy from Chewy (skills-only) or ensure titles+skills in same facet group

## Reusable Code Patterns

### Pattern 1: Bulk Campaign Setup

```python
from linkedin_api import LinkedInAdsClient

client = LinkedInAdsClient()

# List of target companies
companies = [
    ("Stripe", "2135371"),
    ("Atlassian", "1234"),
    ("MongoDB", "5678")
]

for company_name, org_id in companies:
    # Clone template
    new_campaign = client.clone_campaign(
        source_campaign_id=447062324,  # Chewy template
        new_name=f"ABM_{company_name.replace(' ', '')}",
        modifications={}
    )

    # Update targeting
    campaign = client.get_campaign(new_campaign['id'])
    targeting = campaign['targetingCriteria']

    # Update organization
    for facet in targeting['include']['and']:
        if 'urn:li:adTargetingFacet:employers' in facet.get('or', {}):
            facet['or']['urn:li:adTargetingFacet:employers'] = [f'urn:li:organization:{org_id}']

    client.update_campaign(new_campaign['id'], {'targetingCriteria': targeting})

    print(f"✓ {company_name} campaign ready: {new_campaign['id']}")
```

### Pattern 2: Targeting Analysis Script

```python
from linkedin_api import LinkedInAdsClient

client = LinkedInAdsClient()

# Get all ABM campaigns
campaigns = client.search_campaigns()
abm_campaigns = [c for c in campaigns if 'ABM' in c['name']]

for campaign in abm_campaigns:
    # Analyze each
    targeting = campaign.get('targetingCriteria', {})

    # Check for restrictive AND logic
    has_titles = False
    has_skills = False
    same_group = False

    for facet_group in targeting.get('include', {}).get('and', []):
        facets = facet_group.get('or', {})
        group_has_titles = 'urn:li:adTargetingFacet:titles' in facets
        group_has_skills = 'urn:li:adTargetingFacet:skills' in facets

        if group_has_titles:
            has_titles = True
        if group_has_skills:
            has_skills = True
        if group_has_titles and group_has_skills:
            same_group = True

    if has_titles and has_skills and not same_group:
        print(f"⚠️  {campaign['name']}: Restrictive AND logic (titles AND skills separate)")
    elif has_titles and not has_skills:
        print(f"📋 {campaign['name']}: Title-based only")
    elif has_skills and not has_titles:
        print(f"✅ {campaign['name']}: Skills-based (inclusive)")
```

## Decision Trees

### "Should I use title-based or skills-based targeting?"

**Use skills-based (Chewy strategy) when:**
- ✅ ABM campaigns (want to saturate the company)
- ✅ Target audience has non-standard titles
- ✅ Want to catch anyone relevant regardless of title
- ✅ Competitive displacement (target PagerDuty users)

**Use title-based when:**
- ❌ Need very specific seniority (only VPs, only ICs)
- ❌ Extremely narrow use case
- ❌ (Actually, almost never - skills-based is usually better)

**Use hybrid (titles + skills in SAME group) when:**
- ✅ Want precision of titles with flexibility of skills
- ✅ Large companies where titles are standardized

### "Should I pause this campaign?"

**Pause if:**
- ❌ Zero clicks after $50+ spend
- ❌ CPC > $40
- ❌ No creatives serving
- ❌ Zero conversions after $200+ spend

**Keep running if:**
- ✅ Getting conversions (any amount)
- ✅ CPC < $20 and getting clicks
- ✅ Recently launched (<30 days, <$100 spend)

**Review if:**
- ⚠️  Low volume (<10 clicks on $100+ spend)
- ⚠️  CPC $20-40 range

## Key Insights (Learned January 2026)

### Targeting Logic Gotcha

**LinkedIn's AND/OR structure:**
```json
{
  "include": {
    "and": [
      {"or": {"titles": [...]}},     // Group 1
      {"or": {"skills": [...]}}      // Group 2
    ]
  }
}
```

**This means:** (Title1 OR Title2) **AND** (Skill1 OR Skill2)

**To make it inclusive:**
```json
{
  "include": {
    "and": [
      {"or": {
        "titles": [...],
        "skills": [...]       // Same group!
      }}
    ]
  }
}
```

**This means:** Title1 OR Title2 OR Skill1 OR Skill2 (inclusive)

**Or even better - skip titles entirely:**
```json
{
  "include": {
    "and": [
      {"or": {"skills": [...]}},
      {"or": {"seniorities": [...]}},
      {"or": {"jobFunctions": [...]}}
    ]
  }
}
```
